Profile Page

Technical Description

This HTML and CSS code builds a structured and visually appealing profile dashboard. The !DOCTYPE html declaration ensures HTML5 compatibility, while html lang="en" improves accessibility. The head section contains metadata, linked stylesheets (link rel="stylesheet" for styleglobal.css and style.css), and a favicon. The body houses a wrapper div class="wrapper" to center content, with a header containing a logo and a navigation bar (nav with ul class="nav-links"). The main content (main) is divided into three sections: a sidebar (div class="sidebar") displaying user details, skills, and profile information, a central content area (div class="main-content") containing a bio, goals, and experiences, and a right sidebar (div class="right-sidebar") for availability, UN goals, and a contact form. Visual enhancements include skill and goal bars (div class="skill-bar" and div class="goal-bar") with progress indicators. The CSS ensures a responsive design, using flexbox for layout (.container { display: flex; }), background properties (background-size: cover; background-position: center;), and box shadows (box-shadow: 2px 2px 10px rgba(0,0,0,0.1);) for modern styling. The footer contains additional navigation links and credits, enhancing usability and structure.

Accessibility

The profile dashboard boosts accessibility with its use of semantic HTML, making it more navigable through screen readers. Site-Map

Technical Description

The SVG design proposes an interactive navigation nature-illustrated interface by representing buttons through rectangles, labels through texts, and connection through paths. Each node is grouped using g tag and positioned using transform="'translate(x,y)'". Rectangles have rounded corners and come coated in various colors-green for pages, blue for home-for differentiation needs. The text elements are centered inside the rectangles for clear visibility. The nodes are wrapped in clickable links that are made interactive with a xlink:href. Connections between related sections are visually made with path, therefore forming a structured navigation map. That ensures clear, functional, and user-friendly design.

Accessibility

To enhance information accessibility in the SVG navigation, contrast of all text elements was taken into consideration for user-friendly; this will assist users who have vision impairment-related disabilities in reading. Semantic grouping using g tags for easier assistive technology interpretation was done. All links were embedded inside a ;tags to allow keyboard navigation and compatibility with screen readers. Large clickable areas on the buttons are provided for users who have motor impairments. Plus, connections between the different elements assist users in following through easily.

Challenges and Lessons Learned

In the process of implementation, one of the major challenges was in getting the clickable areas in the SVG to work as intended. Several of the links within the a tags were, at one time, not clickable because the anchor elements were not properly nested around the SVG shapes. The particular issue shall be solved by wrapping each of the interactive elements, such as buttons and links, inside the appropriate a tags. Ensuring that both the design and layout of the SVG were in line with the HTML structure presented another challenge, especially the sidebar and content sections. It could lead to a situation where there would be a difference in the ((visual appearance of the two)). This is, however, being dealt with by using the same colors for links and working on making the layout adaptive for other resolutions. In brief, I gained the knowledge to structurally prepare the SVG elements correctly for interactivity, while keeping a common design between HTML and SVG to have a smooth experience for the user.
